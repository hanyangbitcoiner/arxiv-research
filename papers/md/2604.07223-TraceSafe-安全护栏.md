# TraceSafe: Agent安全护栏评估

**论文：** [arXiv:2604.07223](https://arxiv.org/abs/2604.07223)
**作者：** Yen-Shan Chen et al. (CyCraft AI Lab, NTU)
**日期：** 2026-04-08

---

## 核心问题

随着 LLM Agent 演进为**自主工具调用者**，安全漏洞从最终输出转移到**中间执行轨迹**。

---

## 核心贡献

1. **TraceSafe-Bench**：首个评估 Agent 轨迹级安全的基准
2. **12 风险类别**：覆盖提示注入、隐私泄露、幻觉参数、接口不一致
3. **Benign-to-Harmful Editing**：从良性轨迹自动生成有害变体的方法

---

## 风险分类体系

| 类别 | 风险 | 描述 |
|------|------|------|
| **提示注入** | Prompt Injection | 恶意工具定义/输出注入 |
| **隐私泄露** | Privacy Leakage | 用户信息/API Key/数据泄露 |
| **幻觉** | Hallucination | 幻觉工具/参数名/参数值 |
| **接口不一致** | Interface Issues | Schema 不匹配等 |

---

## 关键发现

### 发现 1：结构化能力是瓶颈

护栏效果与结构化任务强相关（ρ=0.79），与语义安全对齐几乎无关。

### 发现 2：架构 > 规模

同系列模型没有单调缩放趋势，代码密集预训练比规模更重要。

### 发现 3：时间稳定性

轨迹越长，风险检测反而越好。

## 伪代码实现

```python
class TraceSafeBenchmark:
    def mutate_trace(self, benign_trace, risk_type, mutation_point):
        mutated = benign_trace.copy()

        if risk_type == 'prompt_injection_in':
            tool = mutated[mutation_point]['tool']
            tool['description'] = tool['description'] + "; pwned"
        elif risk_type == 'api_key_leak':
            mutated[mutation_point]['arguments']['api_key'] = 'sk-fake'
        return mutated
```
