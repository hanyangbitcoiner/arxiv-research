## Cactus: 约束接受Speculative Decoding

**arxiv:** [2604.04987](https://arxiv.org/abs/2604.04987)
**作者:** Yongchang Hao, Lili Mou (University of Alberta)
**日期:** 2026-04-05
**标签:** 解码优化、约束优化

### 核心问题

Speculative Sampling (SpS) 严格强制生成分布匹配验证器分布，但这过于严格——轻微变化（如 top-k 或 temperature 采样）也是可接受的。

**现有方法的问题：**
- SpS 严格匹配分布，导致接受率较低
- TAS 接受更多，但扭曲了验证器分布
- 无法在效率和质量之间取得平衡

### 核心贡献

1. **约束优化框架**：将 speculative sampling 形式化为约束优化问题
2. **Cactus 算法**：保证受控的分布偏离，同时提高接受率
3. **理论基础**：为 TAS 提供新的理论解释

### 背景：Speculative Sampling

```
标准 SpS 流程：
1. Draft 模型生成 m 个 token
2. 验证器模型并行验证
3. 接受的 token 数量由 min(1, q/p) 决定

问题：只有当 q(n) > p(n) 时才能接受
```

### 问题形式化

```
优化目标: max_h min{hn/p(n), 1}
约束:   h ∈ Δ^{|V|-1}  (概率 simplex)
        Df(h || q) ≤ δ  (f-散度约束)
```

**最优解：**
```
h*_i = γ*, if i = n  (采样的 token)
h*_i = (1-γ*) * q(i) / (1-q(n)), otherwise
```

### SpS vs Cactus

| 方面 | SpS | Cactus |
|------|-----|--------|
| 分布匹配 | 严格匹配 | 受控偏离 |
| 接受率 | 较低 | 更高 |
| 输出质量 | 无损 | 可控降级 |
| δ 控制 | 无 | 有 |

### 伪代码实现

```python
class Cactus:
    """
    Cactus: Constrained Acceptance Speculative Sampling

    核心思想：在满足 f-散度约束的前提下，最大化接受率
    """

    def __init__(self, draft_model, verifier_model, delta=0.1):
        self.draft = draft_model
        self.verifier = verifier_model
        self.delta = delta  # 散度约束参数，控制偏离程度

    def decode(self, prompt, max_len, temperature=1.0):
        """
        Cactus 解码主循环
        """
        x = prompt
        generated_tokens = []

        while len(x) < max_len and not self.is_end_of_sequence(x):
            # Step 1: Drafting - 用小模型生成候选 token 序列
            draft_tokens = self.draft.sample(
                x,
                n_tokens=10,  # speculative batch size
                temperature=temperature
            )

            # Step 2: 验证器获取所有 token 的概率分布
            q_full = self.verifier.get_log_probs(x)  # [vocab_size]

            accepted_tokens = []
            rejected = False

            for i, t in enumerate(draft_tokens):
                # Step 3: 获取 draft 和 verifier 对当前 token 的概率
                p_t = math.exp(self.draft.get_log_prob(x, t))
                q_t = math.exp(q_full[t])

                # Step 4: 计算 Cactus 接受概率
                acceptance_prob = self.compute_acceptance_probability(
                    p_t, q_t, self.delta
                )

                # Step 5: 蒙特卡洛采样决定接受或拒绝
                if random.random() < acceptance_prob:
                    accepted_tokens.append(t)
                    x = x + [t]  # 更新生成序列
                else:
                    # Step 6: 拒绝 - 采样恢复 token
                    recover_token = self.sample_recovery_token(
                        q_full, t, p_t, q_t
                    )
                    x = x + [recover_token]
                    rejected = True
                    break

            # Step 7: 如果所有 draft token 都被接受
            if not rejected and len(accepted_tokens) == len(draft_tokens):
                # 添加 bonus token
                bonus_token = self.sample_bonus_token(q_full)
                x = x + [bonus_token]

            generated_tokens.extend(accepted_tokens if not rejected else [])

        return generated_tokens

    def compute_acceptance_probability(self, p, q, delta):
        """
        计算 Cactus 接受概率

        关键洞察：
        - 当 p <= q 时（draft 不如 verifier），接受率与 SpS 一致
        - 当 p > q 时（draft 比 verifier 更自信），Cactus 给予额外提升
        """
        if p <= q:
            # 低置信度 draft：接受率与 SpS 一致
            # SpS: min(1, p/q)
            return min(1.0, p / q)
        else:
            # 高置信度 draft：Cactus 额外提高接受率
            # 求解 gamma 以满足散度约束
            gamma = self.find_optimal_gamma(p, q, delta)

            # 接受率 = gamma * p / q，但不超过 1
            return min(1.0, gamma * p / q)

    def find_optimal_gamma(self, p, q, delta):
        """
        求解最优 gamma 以满足 f-散度约束

        约束条件：
        D_f(h || q) <= delta

        其中 h 是调整后的分布：
        h(n) = gamma * p(n) / C (n = 采样的 token)
        h(i) = (1-gamma) * q(i) / (1-q(n)) (i != n)

        返回满足约束的最大 gamma 值
        """
        # 数值求解：从 q(n) 到 1.0 搜索
        q_n = q  # 简写

        for gamma in torch.linspace(q_n, 1.0, steps=100):
            # 计算调整后分布与原分布的 f-散度
            # 使用 KL 散度作为 f-散度

            # 调整后的 q
            h_n = gamma * p  # 采样 token 的调整后概率
            Z = 1 - q_n + gamma * q_n  # 归一化常数

            # KL(q || h) = sum(q_i * log(q_i / h_i))
            # 只计算采样 token 和恢复 token 的贡献
            kl = q_n * torch.log(q_n / (h_n / Z)) + \
                 (1 - q_n) * torch.log((1 - q_n) / ((1 - gamma) * q_n / Z))

            if kl <= delta:
                return gamma.item()

        return q_n  # fallback：返回 q(n)

    def sample_recovery_token(self, q_full, rejected_token, p, q):
        """
        拒绝后采样恢复 token

        恢复分布 g 应该：
        1. 在 rejected_token 位置概率为 0
        2. 其余 token 概率与 q 成正比
        """
        # 构建恢复分布
        g_probs = torch.zeros_like(q_full)
        for i in range(len(q_full)):
            if i != rejected_token:
                g_probs[i] = math.exp(q_full[i])

        # 归一化
        g_probs = g_probs / g_probs.sum()

        # 采样
        return torch.multinomial(g_probs, 1).item()

    def sample_bonus_token(self, q_full):
        """
        所有 draft token 被接受后，采样 bonus token

        Bonus token 应该增加序列的多样性
        """
        # 直接从 verifier 分布采样
        probs = F.softmax(torch.exp(q_full), dim=-1)
        return torch.multinomial(probs, 1).item()


class CactusTheoretical:
    """
    Cactus 理论分析
    """

    @staticmethod
    def compute_acceptance_rate(spS_p, SpS_q, delta):
        """
        计算理论接受率

        SpS (原始):
        - 接受率 = min(1, q/p) 当 p < q
        - 接受率 = 1 当 p >= q

        Cactus (改进):
        - 当 p <= q: 与 SpS 相同
        - 当 p > q: 接受率 = gamma * p / q，其中 gamma <= 1

        关键洞察：
        Cactus 在 draft 模型过度自信时（p > q）给予额外提升，
        而这在实际应用中很常见。
        """

    @staticmethod
    def divergence_constraint(h, q, f='kl'):
        """
        计算 f-散度 D_f(h || q)

        常用 f-散度：
        - KL 散度: f(t) = t log t
        - reverse KL: f(t) = -log t
        - Pearson: f(t) = (t-1)^2
        """
        if f == 'kl':
            # KL(q || h)
            return sum(q_i * torch.log(q_i / h_i) for i in range(len(q)))
        elif f == 'pearson':
            # Pearson χ^2
            return sum((q_i - h_i)**2 for i in range(len(q)))


class TypicalAcceptanceSampling:
    """
    Typical Acceptance Sampling (TAS) - 基线方法

    TAS 使用熵启发式来接受更多 token，
    但会扭曲验证器分布。
    Cactus 在 TAS 的基础上增加了显式的分布约束。
    """

    def __init__(self, draft_model, verifier_model, threshold=0.5):
        self.draft = draft_model
        self.verifier = verifier_model
        self.threshold = threshold

    def decode(self, prompt, max_len):
        x = prompt
        while len(x) < max_len:
            draft_tokens = self.draft.sample(x, n_tokens=10)
            q = self.verifier.get_log_probs(x)

            # TAS 接受准则：基于熵
            accepted = []
            for t in draft_tokens:
                # 计算局部熵
                entropy = -sum(math.exp(q[i]) * q[i] for i in range(len(q)))

                # 如果熵低于阈值，接受
                if entropy < self.threshold:
                    accepted.append(t)
                    x = x + [t]
                else:
                    break

        return x
```

### 与 TAS 的关系

**TAS 的问题：**
```
H(h, q) = D_KL(h || q) + H(h)
         = mode capture + entropy

TAS 通过最小化交叉熵来提高接受率，
但 entropy 项会导致分布过于确定性，
从而扭曲验证器分布。
```

**Cactus 的改进：**
```
- 使用显式的 f-散度约束
- 不引入额外的 entropy 项
- 接受率提高的同时保持分布受控
```

### 实践建议

1. **δ 选择**：δ 越大接受率越高，但质量损失越大
2. **适用场景**：AR 解码加速，尤其适合 draft-verifier 差异大的情况
3. **注意**：不适用于对分布精确性要求极高的场景（如对话系统）
